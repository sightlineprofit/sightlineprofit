**Non-obvious gotchas:**

- **`SUPABASE_SERVICE_ROLE_KEY` is required for essentially all authenticated/server functionality but is NOT in `.env`.** Every TanStack server function (`src/lib/*.functions.ts`) uses the admin client in `src/integrations/supabase/client.server.ts`, which throws `Missing Supabase environment variable(s): SUPABASE_SERVICE_ROLE_KEY` when it's unset. Symptoms without it: the `/register` page gets stuck on a "Fetching pricingâ€¦ try again in a moment." toast (the founding-price quote server fn fails), and firm creation in `/post-auth`, billing, admin, team, etc. all fail. Client-side Supabase auth (login/signup) works without it because it uses the public anon key already in `.env`. To exercise the authenticated app end to end, set `SUPABASE_SERVICE_ROLE_KEY` (server-only) as an env var/secret.
- The remote Supabase project **requires email confirmation**: `supabase.auth.signUp` returns `confirmation_sent_at` and logging in before confirming returns `email_not_confirmed`. Plan for email confirmation (or admin-created users via the service role key) when testing login â†’ dashboard.
